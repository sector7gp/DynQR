## 🎯 Flujo de Usuario


## 🎨 Características Implementadas

### ✅ QR Dinámico
- Genera nuevas imágenes cada 30 segundos
- Token único por sesión
- Codifica URL con token

### ✅ Sistema de Reservas
- Iframe de Acuity Scheduling embebido
- Acceso protegido por token
- Usuario puede hacer reservas

### ✅ Validación de Acceso
- Tokens generados cada 30s
- Backend valida en tiempo real
- Único token válido a la vez

### ✅ Pantalla de Error
- Amigable y clara
- Anima al usuario a escanear QR válido
- Muestra razón del rechazo

## 🔐 Seguridad

✅ **Tokens únicos**
- Cada 30 segundos: nuevo UUID
- Anterior se invalida automáticamente
- Solo 1 token válido a la vez

✅ **Validación en Backend**
- No se confía en cliente
- Cada acceso se valida en servidor
- Imposible falsificar acceso

✅ **Sin exposición de URL**
- iframe de Acuity no es accesible directamente
- Solo funciona con token válido
- Protección automática
