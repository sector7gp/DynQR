🎯 CÓMO FUNCIONA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. USUARIO VE QR
   → Página principal muestra código QR dinámico
   → Contador visible: 30, 29, 28... 0 (luego reinicia a 30)
   → QR contiene URL con token único

2. USUARIO ESCANEA QR
   → Escanea con su teléfono
   → URL se abre: /reservations?token=UNIQUE_TOKEN
   → Token se valida en backend

3. SI ACCESO VÁLIDO (token correcto)
   → Muestra página de reservas
   → Embebido: iframe de Acuity Scheduling
   → Usuario puede hacer su reserva

4. SI ACCESO INVÁLIDO (token expirado o falta)
   → Muestra página de error amigable
   → Mensaje: "Acceso Denegado"
   → Invita a escanear QR válido
